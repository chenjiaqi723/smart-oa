# SmartOA 数据库设计

## 1. 数据库说明

数据库名称：smart_oa

字符集：utf8mb4

排序规则：utf8mb4_general_ci

---

## 2. 第一阶段数据表

### employee（员工表）

用于存储员工基本信息。

| 字段名           | 类型          | 是否为空 | 默认值               | 说明              |
|---------------|--------------|------|-------------------|-----------------|
| id            | BIGINT       | 否    |                   | 主键（自增）          |
| emp_no        | VARCHAR(20)  | 否    |                   | 员工编号            |
| emp_name      | VARCHAR(50)  | 否    |                   | 员工姓名            |
| gender        | TINYINT      | 否    |                   | 性别（1男、2女）       |
| phone         | VARCHAR(20)  | 是    |                   | 手机号             |
| email         | VARCHAR(100) | 是    |                   | 邮箱              |
| department_id | BIGINT       | 是    |                   | 所属部门            |
| hire_date     | DATE         | 是    |                   | 入职日期            |
| status        | TINYINT      | 否    | 1                 | 状态（1在职、0离职）     |
| is_deleted    | TINYINT      | 否    | 0                 | 删除状态（0未删除、1已删除） |
| create_time   | DATETIME     | 否    | CURRENT_TIMESTAMP | 创建时间            |
| update_time   | DATETIME     | 是    | CURRENT_TIMESTAMP | 更新时间（修改时自动更新时间） |

---

### department（部门表）

用于存储企业部门信息。

| 字段名         | 类型          | 是否为空 | 默认值               | 说明                    |
|-------------|-------------|------|-------------------|:----------------------|
| id          | BIGINT      | 否    |                   | 主键（自增）                |
| dept_no     | VARCHAR(20) | 否    |                   | 部门编号                  |
| dept_name   | VARCHAR(50) | 否    |                   | 部门名称                  |
| parent_id   | BIGINT      | 是    |                   | 上级部门                  |
| manager_id  | BIGINT      | 是    |                   | 部门负责人 （关联employee.id） |
| status      | TINYINT     | 否    | 1                 | 状态（1启用、0停用）           |
| is_deleted  | TINYINT     | 否    | 0                 | 删除状态（0未删除、1已删除）       |
| create_time | DATETIME    | 否    | CURRENT_TIMESTAMP | 创建时间                  |
| update_time | DATETIME    | 是    | CURRENT_TIMESTAMP | 更新时间（修改时自动更新时间）       |

---

### role（角色表）

用于存储系统角色信息。

主要字段：

* 角色编号
* 角色名称
* 角色编码
* 角色描述
* 角色状态
* 创建时间
* 更新时间

---

### user_role（用户角色关联表）

用于维护员工与角色之间的关联关系，一个员工可以拥有多个角色，一个角色也可以分配给多个员工。

主要字段：

* 主键 ID
* 员工 ID
* 角色 ID

---
